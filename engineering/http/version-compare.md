# http1.0、http1.1、http2 版本对比

## http 1.0

只能进行短连接

## http 1.1

### 优点

- 持久连接，connection: keep-alive
- 管道机制，同一个TCP连接里，客户端可以同时发送多个请求

### 缺陷

- 高延迟
    队头阻塞(Head-Of-Line Blocking)
- 无状态
    头部开销巨大，每次请求都要发送一个新的请求头
- 明文传输
    不安全
- 不支持服务推送

## http 2.0

### 优点

- 多路复用
- 二进制分帧
- 头部压缩，降低开销(可以缓存头部信息)
- 服务器推送

### 缺陷

- TCP + TLS 建立连接的延时
- TCP 的队头阻塞并没有彻底解决
- 多路复用导致服务器压力上升，容易超时
